# Project scaffold + Projection engine

## Parent

`.scratch/finplan-core/PRD.md`

## What to build

Set up the web application with routing and empty placeholder screens for the three primary views: Projection chart, Account/ExternalParty list, and Schedule list.

Implement the Projection engine as a pure function. It accepts a set of Accounts (with seed balances and Rates), a set of Schedules, an optional Scenario override set, and a date range. It returns a time series of per-Account balances — one balance entry per Account per period (daily or monthly resolution TBD by implementor).

The engine must handle:
- Fixed-amount Transfers generated by Schedules on their correct calendar dates (respecting biweekly vs semi-monthly distinctions)
- Rate compounding applied to each Account's balance each period
- Amortizing Account termination: once the balance reaches zero, inbound Schedules stop firing for that Account
- Scenario overrides: use overridden Schedule amounts/Accounts in place of Baseline values where specified

Wire a hardcoded dataset (two Accounts, two Schedules, one Rate) through the engine and render it on the chart to prove the full vertical path works end-to-end.

Write a full test suite for the Projection engine covering the scenarios listed in the PRD Testing Decisions section.

## Acceptance criteria

- [ ] Web app renders in the browser with navigation between the three primary views
- [ ] Projection engine is implemented as a pure function with no UI or storage dependencies
- [ ] Engine correctly computes balances for a multi-Account, multi-Schedule dataset over a 12-month range
- [ ] Biweekly and semi-monthly frequencies produce distinct cash flow patterns (26 vs 24 occurrences per year)
- [ ] Rate compounds correctly on an Account's running balance each period
- [ ] An amortizing Account's balance terminates at zero and inbound Schedules cease
- [ ] A Scenario override set produces a divergent balance series from the Baseline
- [ ] A one-time Schedule (frequency: once) fires exactly once on the correct date
- [ ] All engine tests pass
- [ ] Hardcoded dataset renders as a chart in the browser
